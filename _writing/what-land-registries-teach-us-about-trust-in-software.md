Building a cadastral system in a context where land tenure is contested reveals something fundamental: the hardest technical problems are social problems in disguise. The database schema matters less than the political agreements it encodes. Software designed to establish trust must first earn it from the institutions it serves.

## The illusion of the technical solution

When engineers approach land registration, they tend to think in schemas, APIs, and access controls. But a cadastral system in sub-Saharan Africa is not primarily a technical artifact—it is a political one. The boundaries it draws are not simply geographic; they encode power, history, and contested claims.

## Trust as infrastructure

Software that mediates trust must itself be trustworthy. This means transparency in how decisions are made, auditability in how records change, and humility in acknowledging what the system cannot resolve. No database can adjudicate a land dispute that predates the colonial era.

## Lessons for all systems

The principles that govern cadastral software apply broadly: every system that mediates between parties must earn legitimacy before it can function. Technology alone is never sufficient.
